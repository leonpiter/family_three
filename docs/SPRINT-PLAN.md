# Спринт-план: Семейное генеалогическое древо

Проект: веб-сервис семейного древа для одной семьи — бесконечная доска с круглыми фото-нодами родственников (DnD, связи «родитель—ребёнок» и «супруги»), правый сайдбар с профилем человека и фотоальбомом. Регистрация открыта, доступ — после одобрения админом.

Стек: React 19 + Vite 7 + TypeScript + Tailwind 4 + @xyflow/react 12 (доска) + zustand; бэкенд — Supabase free tier (Auth + Postgres/RLS + private Storage); хостинг фронта — GitHub Pages. Технические детали — в [ARCHITECTURE.md](ARCHITECTURE.md).

Оценки задач: **S** — до часа, **M** — 1–3 часа, **L** — 3–6 часов (условно, разработка агентная).

## Спринт 1 — «Вход в дом»: каркас, аутентификация, одобрение

**Цель спринта:** развёрнутое на GitHub Pages приложение, в котором можно зарегистрироваться, подтвердить email, дождаться одобрения админа и попасть внутрь. Безопасность работает на уровне БД, а не только UI.

**Результат для пользователя:** «Я захожу на сайт с телефона, регистрируюсь, админ одобряет меня — и я внутри».

### Пользовательские истории

| ID | История | Критерии приёмки |
|---|---|---|
| S1.1 | Как гость, я могу зарегистрироваться по email и паролю | Форма /register с именем, email, паролем; после отправки — экран «подтвердите почту»; письмо приходит; ссылка из письма возвращает на сайт уже залогиненным |
| S1.2 | Как новый пользователь, я вижу статус «ожидает одобрения» и не вижу данных семьи | После подтверждения email — страница /pending; прямые запросы к persons/photos через API возвращают пусто (RLS), не только скрыты в UI |
| S1.3 | Как админ, я вижу список заявок и одобряю/отклоняю их | Страница /admin видна только админу; кнопки «Одобрить»/«Отклонить» меняют статус; одобренный пользователь после обновления страницы попадает внутрь |
| S1.4 | Как одобренный пользователь, я захожу и выхожу из системы | /login работает, ошибки («неверный пароль») по-русски; кнопка «Выйти» в шапке |

### Definition of Done спринта

- Полный e2e вручную на проде: регистрация второго тестового email → письмо → pending → одобрение из /admin → доступ.
- RLS проверен запросами в обход UI (SQL Editor / curl с anon key): pending не видит ничего, кроме своего профиля.
- Все строки UI — русские, из `strings.ts`; ошибки показываются тостами.
- README: как поднять локально, как бутстрапить админа.

### Риски спринта

- Письма от встроенного SMTP Supabase могут попадать в спам / лимит единиц в час — для теста хватит, при проблемах подключить Resend (перенос в Спринт 4).
- Ошибка в `base` vite-конфига или Redirect URLs ломает подтверждение email на проде — поэтому деплой в первом спринте, а не в последнем.

## Спринт 3 — «Альбом»: сайдбар, фотографии, аватары

**Цель спринта:** клик по родственнику открывает правый сайдбар с его историей и фотоальбомом; фото грузятся с телефона, сжимаются, защищены от постороннего доступа.

**Результат для пользователя:** «Я открыл карточку бабушки, прочитал её историю и загрузил 10 старых фото с подписями».

### Пользовательские истории

| ID | История | Критерии приёмки |
|---|---|---|
| S3.1 | Клик по кружочку открывает карточку человека | Сайдбар ~380px справа: ФИО, девичья фамилия, годы, место рождения, биография; закрытие — ×, Esc, клик по полотну |
| S3.2 | Я редактирую информацию о человеке | Кнопка «Редактировать» → форма → сохранение с тостом; если карточка привязана к моему аккаунту — бейдж «Это вы» |
| S3.3 | Я загружаю фото в альбом человека | Drag-n-drop и выбор файлов (multiple); прогресс; тяжёлый JPEG с телефона превращается в webp ≤ 0.4 МБ + thumb |
| S3.4 | Я смотрю альбом и подписываю фото | Грид превью в 3 колонки; лайтбокс с клавиатурой/свайпами; подпись редактируется |
| S3.5 | Я выбираю аватар для кружочка на доске | Кнопка «Сделать аватаром» в лайтбоксе; кружок на доске показывает фото вместо инициалов |
| S3.6 | Фото семьи не видны посторонним | Прямой URL файла без подписи → 403/400; signed URL работает и истекает |

### Definition of Done спринта

- 10 тяжёлых фото с телефона (включая HEIC/большие JPEG) загружены: в Storage — webp ≤ 0.4 МБ + thumb ≤ 0.05 МБ каждая.
- S3.6 проверен напрямую: URL из bucket без подписи не открывается ни анонимно, ни pending-аккаунтом.
- Доска и альбом тянут только thumb'ы; полноразмер — только в лайтбоксе (проверить network-вкладкой).

### Риски спринта

- HEIC с iPhone: browser-image-compression декодирует не во всех браузерах — проверить на реальном iPhone; при проблеме — подсказка «выберите JPEG», конвертация — в бэклог.
- Egress free tier (5 ГБ/мес): без thumb-first и кэша подписей улетит быстро — DoD это ловит.

## Спринт 4 — «Новоселье»: полировка, прод, передача семье

**Цель спринта:** сервис готов к передаче реальным родственникам — понятные тексты, обработанные ошибки, админ-инструменты, страховка от засыпания и потери данных.

**Результат для пользователя:** «Я разослал ссылку родне, все зарегистрировались и начали заполнять древо без моей помощи».

### Пользовательские истории

| ID | История | Критерии приёмки |
|---|---|---|
| S4.1 | Как админ, я привязываю аккаунт к персоне на дереве | В /admin выпадающий список персон у каждого профиля; после привязки у пользователя в карточке бейдж «Это вы» |
| S4.2 | Как админ, я удаляю персону целиком | Кнопка удаления (только админ) с подтверждением; каскадно уходят связи и фото (включая storage-объекты) |
| S4.3 | Я понимаю, что происходит, при любой ошибке | Пустые состояния («На доске пока никого — добавьте первого родственника»), тосты ошибок по-русски, спиннеры загрузки |
| S4.4 | Сервис не «умирает» от простоя | keepalive-cron зелёный две недели подряд; сайт открывается после недели тишины |

| 4.6 | Финальный e2e на проде с двух устройств (десктоп + телефон) | M | Чек-лист ниже пройден |
| 4.7 | (опционально, если остался запас) Realtime-подписка на persons/relationships | L | Изменение с одного устройства видно на втором без F5 |

### Финальный e2e-чек-лист (прод)

1. Регистрация с телефона нового email → письмо → подтверждение → pending.
2. Одобрение с десктопа из /admin → телефон после обновления внутри.
3. С телефона: добавить персону, подвинуть, связать, загрузить 2 фото, поставить аватар.
4. С десктопа: всё видно; отредактировать биографию; привязать аккаунт к персоне.
5. Выйти → анонимно и pending-аккаунтом убедиться: данные и фото недоступны (в т.ч. прямыми запросами).

### Definition of Done проекта

- Все 4 спринтовых DoD закрыты; e2e-чек-лист пройден на проде.
- В репозитории нет секретов (только публичные URL/anon key в variables).
- README позволяет пересоздать всё с нуля без участия автора.
